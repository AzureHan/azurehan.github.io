# Redis 笔记

Redis 是键值型数据库，支持数据类型有：字符串（String）、列表（List）、集合（Set）、有序集合（Sorted Set）、哈希表（Hash）。

# 1. 数据类型

# 1.1 字符串（String）

# 常用命令

SET ：创建 / 修改 键的值

STRLEN ：返回字符串值的长度

GET / MGET ：返回 键的值

INCR / DECR / INCRBY / DECRBY ：数值 自增 或 自减

# 应用场景

  • 字符串缓存
  • 计数器

# 1.2 列表（List）

# 常用命令

LPUSH / RPUSH / LSET ：创建 或 修改 成员

LLEN ：返回列表的长度

LINDEX / LRANGE / LPOP / RPOP ：返回 或 弹出 成员

LREM / LTRIM ：移除成员

# 应用场景

  • 任务队列
  • 可分页的按最近时间排序的评论列表
  • 关注列表 / 粉丝列表
  • 最近访问记录

# 1.3 集合（Set）

# 常用命令

SADD ：创建成员。

SCARD key ：返回成员数量

SINTER / SUNION / SDIFF 返回集合之间的 交集 / 并集 / 差集

SMEMBERS / SRANDMEMBER / SPOP key ：返回 / 弹出 成员

SREM ：移除成员

# 应用场景

  • 共同好友 / 共同关注
  • 记录文章已读的用户
  • 获取某段时间所有数据排重值

# 1.4 有序集合（Sorted Set）

# 常用命令

ZADD ：创建成员

ZCOUNT / ZCARD ：返回成员数量

ZRANGE / ZREVRANGE / ZRANGEBYSCORE / ZREVRANGEBYSCORE ：返回成员

ZREM / ZREMBYRANK / ZREMBYSCORE ：移除成员

# 应用场景

  • 排行榜 / 投票榜
  • 延时任务队列 / 优先级任务队列
  • 记录过期时间

# 1.5 哈希表（Hash）

# 常用命令

HSET / HMSET ：设置值

HEXISTS ：检查域存在

HLEN / HSTRLEN ：返回 域 或 值的字符 数量

HGET / HMGET / HGETALL / HKEYS / HVALS ：返回域 / 值

HINCRBY / HINCRBYFLOAT ：数值增量

HDEL ：删除域

# 应用场景

  • 商品详情 / 个人信息
  • 一天的用户活跃时间

# 2. 数据持久化

有两种不同的持久化方法可以将数据存储在磁盘中，一种叫快照 RDB ，另一种叫只追加文件 AOF 。RDB 用于生成某个时间点的备份信息，并且会对当前的 key-value 进行编码存储到 RDB 文件中，恢复时是将快照文件直接读到内存里。AOF 以执行日志的形式，记录服务器所有的写请求存储到 AOF 文件中，在服务重启的时候通过回放执行命令请求来恢复原有的数据。

# 2.1 RDB

RDB 一般只用作后备用途，建议只在 Slave 上持久化 RDB 文件，而且只要 15 分钟备份一次就够了，只保留 save 900 1 这条规则。由于「快照」的特性，无法增量更新数据，只能保存新的一份 RDB 文件。

# 2.2 AOF

AOF 文件记录的是 Redis 通信协 的写请求命令，RESP (REdis Serialization Protocol) 的特点是：易实现、解析快、易理解。客户端将请求以 RESP 形式发送到服务端，服务端执行完成后将请求追加写入 AOF 文件中。写入频率对应配置项 appendfsync 的两种选项： always 和 everysec ，后者效率较高但数据完整性较差。

# 2.3 对比 RDB 与 AOF

模式 效率 数据完整性 恢复速度 文件体积
RDB 高 差 快 小
AOF 差 / 较差 好 / 较好 慢 大

Redis 4.0 开始允许使用 RDB & AOF 混合持久化的方式结合了两者的优点通过 aof-use-rdb-preamble 配置项可以打开混合开关。这种情况下如果重启 Redis ，因为 AOF 数据更加完整，所以会优先从 AOF 中恢复数据。

除了使用 RDB + AOF 数据持久化保证数据可用性，也可以使用 RDB + 主从复制 保证可用性，同时还能获得较高的效率。

# 3. 淘汰策略

  • volatile-lru ：从已设置过期时间的数据集（server.db[i].expires）中挑选最近最少使用的数据淘汰
  • volatile-ttl ：从已设置过期时间的数据集（server.db[i].expires）中挑选将要过期的数据淘汰
  • volatile-random ：从已设置过期时间的数据集（server.db[i].expires）中任意选择数据淘汰
  • allkeys-lru ：从数据集（server.db[i].dict）中挑选最近最少使用的数据淘汰
  • allkeys-random ：从数据集（server.db[i].dict）中任意选择数据淘汰
  • no-eviction ：禁止剔除数据
