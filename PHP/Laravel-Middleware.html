# Laravel 中间件

# 1. 实现原理

# 2. 源码剖析

基 于 源 码 的 简 化 示 例 代 码

/* 实现中间件所依赖的「 管道 」类 */
class Pipeline
{
    public function send($passable)
    {
        $this->passable = $passable;

        return $this;
    }

    public function through($pipes)
    {
        $this->pipes = $pipes;

        return $this;
    }

    public function then(Closure $destination)
    {
    	/* 函数 array_reduce */
    	/* 参数一 : 需要迭代处理的数组 */
    	/* 参数二 : 回调函数 */
    	/* 参数三 : 初始值 */
        $pipeline = array_reduce(
            array_reverse($this->pipes),

            /* 匿名函数 */
            /* 参数一 : 携带上次迭代里的结果 ; 第一次迭代是「 初始值 」 */
            /* 参数二 : 携带了本次迭代的数组元素 */
            function ($stack, $pipe) {
	            return function ($passable) use ($stack, $pipe) {
	            	/* 结果对应「 中间件 」的 handle() */
	                return $pipe($passable, $stack);
	            };
	        },

	        $destination
        );

        return $pipeline($this->passable);
    }
}

/* 数组元素对应「 中间件 」的 handle() */
$middleware = [
    function ($request, Closure $next) {
	    $request->count += 1;
	    return $next($request);
	},
    function ($request, Closure $next) {
	    $request->count -= 1;
	    return $next($request);
	},
];

/* 路由最终指向的可执行内容 */
$destination = function ($request) {
    return view('welcome');
};

/* 类实例化执行后得到响应 */
$response = new Pipeline()
	->send($request)
	->through($middleware)
	->then($destination);

# 2.1 问题点

# 2.1.1 函数 array_resereve() 的作用

源码中了解到定义中间件数组是有执行顺序的。

Illuminate\Routing\Route::class

/**
* Sort the given middleware by priority.
*
* @param  \Illuminate\Support\Collection  $middlewares
* @return array
*/
protected function sortMiddleware(Collection $middlewares)
{
	return (new SortedMiddleware($this->middlewarePriority, $middlewares))->all();
}

但按照源码来执行结果是与数组的顺序相反的，因为 array_reduce 的回调函数返回的是一个匿名函数。而匿名函数里套匿名函数的情况与递归函数相似，再而递归可由栈来实现，因此最先迭代的反而是最后执行的。

等 价 于 源 码 简 化 示 例 的 代 码

$response = function ($request) {
    $request->count -= 1;
    function ($request) {
	    $request->count += 1;
	    return view('welcome');
	};
    return $request;
}

# 2.1.2 变量 $destination 的详情

示例中明显地看到 $destination 的值是 Laravel 初始化时的根路由匿名函数，因此这里简化了关联路由选择的详情。事实上示例中的中间件是全局中间件，而简化的部分还有一次路由级的中间件的执行。
