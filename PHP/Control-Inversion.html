# PHP 控制反转

# 1. 简介

对象之间难免会有各种各样的依赖关系，如果我们的代码直接依赖于具体的实现，那么就是一种强耦合，从而降低了系统的灵活性，为了解耦，我们的代码应该依赖接口，至于具体的实现，则通过第三方注入进去，这里的第三方通常就是我们常说的容器。因为在这个过程中，具体实现的控制权从我们的代码转移到了容器，所以称之为控制反转。

控制反转可以给程序的代码解耦，将原本在内部依赖的功能由外部注入。很多时候我们为了提高代码的扩展性和维护性，在编写类时依赖的是接口或抽象类，而并不是一个具体的实现类。当内部依赖的功能需要更改，就只需要替换注入的部分即可实现，不需要维护程序代码。

# 2. 术语

# 2.1 IoC ( Inversion of Control / 控制反转 )

  • 依赖关系的转移
  • 依赖抽象而非实例

# 2.2 DI ( Dependency Injection / 依赖注入 )

  • 不必自己在代码中维护对象的依赖
  • 容器自动根据配置，将依赖注入指定对象

# 2.3 AOP ( Aspect-oriented programming / 面向方面编程 )

  • 无需修改任何一行程序代码，将功能加入至原先的应用程序中，也可以在不修改任何程序的情况下移除

# 3. 实现依赖注入容器

程序首先注册依赖，在需要时从容器中取出依赖。

适用于 ( 简单绑定、绑定一个单例、绑定实例、绑定初始数据、绑定接口到实现、上下文绑定 ) 。

容器设计魔术方法 Setter 和 Getter ，设计方法 Build ， GetDependencies 和 GetDefaultValue 。

# 3.1 魔术方法 Setter

  1. 直接保存传入的键值对到容器内部的注册列表

# 3.2 魔术方法 Getter

  1. 根据传入的键名在注册列表取出值并调用方法 Build 并返回结果

# 3.3 方法 Build

  1. 通过反射获取目标的构造函数、如果没有就直接实例化返回结果，否则反射获取构造函数的参数
  2. 使用参数调用方法 GetDependencies 获取依赖
  3. 使用依赖通过反射实例化目标并且返回结果

# 3.4 方法 Get Dependencies

  1. 遍历传入的反射参数
  2. 如果参数是 Class 就调用方法 Build 获取实例，否则调用方法 GetDefaultValue 获取默认值、最后加入依赖数组
  3. 返回依赖数组结果

# 3.5 方法 Get Default Value

  1. 反射获取参数默认值并且返回结果

# 4. 实现自动依赖注入

# 需要注入依赖的控制器动作
class Controller
{
    // 依赖于Request对象
    public function show(Request $request)
    {
        var_dump($request);
    }
}

# 自动依赖注入
$class = new ReflectionClass('Controller');
$method = $class->getMethod('show');
// 全部参数类型
$arguments = [];
foreach($method->getParameters() as $parameter) {
    $type = $parameter->getClass()->getName();
    $arguments[] = $app->make($type);
}
// 调用方法，同时注入依赖对象
$method->invokeArgs($class->newInstance(), $arguments);
