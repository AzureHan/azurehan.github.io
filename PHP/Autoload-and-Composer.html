# 5. Composer 原理

Composer 是 PHP 的一个依赖管理工具。它允许你申明项目所依赖的代码库，它会在你的项目中为你安装他们。

# 5.1 使用 Composer

Composer 不是一个包管理器。是的，它涉及 "packages" 和 "libraries"，但它在每个项目的基础上进行管理，在你项目的某个目录中（例如 vendor）进行安装。默认情况下它不会在全局安装任何东西。因此，这仅仅是一个依赖管理。

一旦使用 Composer ，只需要在项目的引导文件的代码中引入一个 PHP 文件即可。此文件返回一个「自加载器」，但更重要的是它本身已为自加载依赖的代码库方法进行注册。

require 'vendor/autoload.php';

实际上在程序运行之前、Composer 依赖代码库之时已经处理好各个代码库的目录和命名空间「关系」，程序运行时只需要检索类名根据加载方法就能找到依赖文件。

生 成 自 加 载 器 部 分 源 码

/* vendor/composer/autoload_real.php */
/* function getLoader */

/**
 * 实例化「自加载器」类
 */
$loader = new \Composer\Autoload\ClassLoader();

/**
 * 初始化已处理好的「关系」，是用于找到依赖文件的根据
 */
call_user_func(\Composer\Autoload\ComposerStaticInit::getInitializer($loader));

/**
 * 注册自加载方法
 */
$loader->register(true);

# 5.2 Composer 自加载器

自加载器兼容 PSR-0 与 PSR-4 ；为 PHP 5.6 及以上版本使用「静态」方式初始化、PHP 5.6 以下和 HHVM使用「一般」方式初始化。

# 5.2.1 注册自加载方法

具 体 实 现 代 码

/* vendor/composer/ClassLoder.php */

class ClassLoader
{
    /**
     * Registers this instance as an autoloader.
     *
     * @param bool $prepend Whether to prepend the autoloader or not
     */
    public function register($prepend = false)
    {
        spl_autoload_register(array($this, 'loadClass'), true, $prepend);
    }

    /**
     * Loads the given class or interface.
     *
     * @param  string    $class The name of the class
     * @return bool|null True if loaded, null otherwise
     */
    public function loadClass($class)
    {
        if ($file = $this->findFile($class)) {
            includeFile($file);

            return true;
        }
    }
}

/**
 * Scope isolated include.
 *
 * Prevents access to $this/self from included files.
 */
function includeFile($file)
{
    include $file;
}

# 5.2.2 依赖「关系」

文中 5.1 中的「关系」在运行时存储在 ClassLoder 类的数个列表中，由文中 5.2.1 自加载方法所需要。

数 个 「 关 系 」 列 表

/* vendor/composer/ClassLoder.php */

class ClassLoader
{
    // PSR-4
    private $prefixLengthsPsr4 = array();
    private $prefixDirsPsr4 = array();
    private $fallbackDirsPsr4 = array();

    // PSR-0
    private $prefixesPsr0 = array();
    private $fallbackDirsPsr0 = array();

    private $useIncludePath = false;
    private $classMap = array();
    private $classMapAuthoritative = false;
    private $missingClasses = array();
    private $apcuPrefix;
}

它们辅助 ClassLoder 类的 findFile 加 findFileWithExtension 方法找到对应类所应该依赖的文件。

# 5.2.2.1 检索 PSR-4 列表

这个列表对应了顶级命名空间和根目录，即 prefixDirsPsr4 类成员。

成 员 部 分 数 组 内 容

/* vendor/composer/autoload_static.php */

public static $prefixDirsPsr4 = array (
    'Illuminate\\Notifications\\' => array (
        0 => __DIR__ . '/..' . '/laravel/slack-notification-channel/src',
        1 => __DIR__ . '/..' . '/laravel/nexmo-notification-channel/src',
    ),
    'Illuminate\\' => array (
        0 => __DIR__ . '/..' . '/laravel/framework/src/Illuminate',
    ),
);

对所依赖的类名中提取其命名空间，然后在列表中循环，找到对应的根目录，拼接上类名字加扩展名，最后判断文件是否存在，最终返回文件路径或 false 。

此时源码中有个问题是 prefixLengthsPsr4 类成员的用处变小 ？

具 体 实 现 部 分 代 码

/* vendor/composer/ClassLoder.php */

class ClassLoader
{
    private function findFileWithExtension($class, $ext)
    {
        $logicalPathPsr4 = strtr($class, '\\', DIRECTORY_SEPARATOR) . $ext;

        $subPath = $class;
        while (false !== $lastPos = strrpos($subPath, '\\')) {
            $subPath = substr($subPath, 0, $lastPos);
            $search = $subPath . '\\';
            if (isset($this->prefixDirsPsr4[$search])) {
                $pathEnd = DIRECTORY_SEPARATOR . substr($logicalPathPsr4, $lastPos + 1);
                foreach ($this->prefixDirsPsr4[$search] as $dir) {
                    if (file_exists($file = $dir . $pathEnd)) {
                        return $file;
                    }
                }
            }
        }

        return false;
    }
}
# 5.2.2.2 检索 PSR-0 列表

这个列表对应了顶级命名空间和根目录，即 prefixesPsr0 类成员。

成 员 部 分 数 组 内 容

/* vendor/composer/autoload_static.php */

public static $prefixesPsr0 = array (
    'P' => array (
        'Prophecy\\' => array (
            0 => __DIR__ . '/..' . '/phpspec/prophecy/src',
        ),
        'Parsedown' => array (
            0 => __DIR__ . '/..' . '/erusev/parsedown',
        ),
    ),
    'M' => array (
        'Mockery' => array (
            0 => __DIR__ . '/..' . '/mockery/mockery/library',
        ),
    ),
);

PSR-0 的具体实现区别在于与 PSR-4 之间的区别 ( 1. 下划线符号作用 2. 命名空间对应文件目录 ) 。

具 体 实 现 部 分 代 码

/* vendor/composer/ClassLoder.php */

class ClassLoader
{
    private function findFileWithExtension($class, $ext)
    {
        $first = $class[0];
        $logicalPathPsr0 = '类似于文中 5.2.2.1 $logicalPathPsr4 的值';
        
        if (isset($this->prefixesPsr0[$first])) {
            foreach ($this->prefixesPsr0[$first] as $prefix => $dirs) {
                if (0 === strpos($class, $prefix)) {
                    foreach ($dirs as $dir) {
                        if (file_exists($file = $dir . DIRECTORY_SEPARATOR . $logicalPathPsr0)) {
                            return $file;
                        }
                    }
                }
            }
        }

        return false;
    }
}
# 5.2.2.3 检索 ClassMap 列表

这个列表直接对应了完整类名和具体依赖文件，即 classMap 类成员。这个列表是优化性质的，因此检索 ClassMap 列表的顺序比 PSR-4 和 PSR-0 都要优先。

它可以由 Composer 命令生成 :

$ composer dump-autoload

因为其优化性质，列表内容也「简单粗暴」，是一个数量庞大的数组。

成 员 部 分 数 组 内 容

/* vendor/composer/autoload_static.php */

public static $classMap = array (
    'App\\Console\\Kernel' => __DIR__ . '/../..' . '/app/Console/Kernel.php',
    'App\\Exceptions\\Handler' => __DIR__ . '/../..' . '/app/Exceptions/Handler.php',
    'App\\Extensions\\Http\\Request' => __DIR__ . '/../..' . '/app/Extensions/Http/Request.php',
);

源码中 findFile 在 findFileWithExtension 之前运行，所以也验证优先级比 PSR-4 和 PSR-0 高。

具 体 实 现 部 分 代 码

/* vendor/composer/ClassLoder.php */

class ClassLoader
{
    public function findFile($class)
    {
        if (isset($this->classMap[$class])) {
            return $this->classMap[$class];
        }
    }
}
