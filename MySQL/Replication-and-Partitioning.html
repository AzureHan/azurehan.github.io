# MySQL 主从复制、分区 和 分库分表

# 1. 简介

中文 英文 维度 对象 简介 特性
主从复制 Replication 横向 数据库实例 将 MySQL 实例 复制成多份，有主从之分 高可用性
读写分离 Read-Write Splitting 横向 数据库实例 基于主从复制架构部署 一主多从 ，主写从读 ，由程序控制连接对象 读高并发性
分区 Horizontal Partitionning / Sharding 横向 数据表文件 配置 MySQL 按预定的规则分割一个数据表的一份 存储文件 成为 多份 读高并发性
分库分表 Horizontal Partitioning / Sharding 横向 数据库 / 表 手动创建多个 库 / 表 来达到分区表的 相同目的 ，更具有 可控性 读高并发性
分库分表 Vertical Partitioning 纵向 数据库 / 表 分割 单个 库 / 表 的结构成 多个 ，分别部署在不同的 硬件 上 写高并发性

# 2. 主从复制

主从架构本来就是一种 高可用性 解决方案，而不是高并发解决方案。

# 2.1 实现主从复制

# 涉及的软件
软件名称 版本
Ubuntu 16.04
MySQL 5.7
# 主机网络
主机名称 网络地址 MySQL 主机序号
Master 192.168.56.30 1
Slave-1 192.168.56.31 2

# 主服务器配置

$ sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

server-id = 1
log_bin = /var/log/mysql/mysql-bin.log

$ sudo systemctl restart mysql

$ mysql -u root -p

mysql> CREATE USER 'myslave'@'192.168.%' IDENTIFIED BY 'mysecret';

mysql> GRANT ALL PRIVILEGES ON *.* TO 'myslave'@'192.168.%' IDENTIFIED BY 'mysecret';

mysql> GRANT REPLICATION SLAVE ON *.* TO 'myslave'@'192.168.%' IDENTIFIED BY 'mysecret';

mysq> SHOW MASTER STATUS;

# 从服务器配置

$ sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

server-id = 2
log_bin = /var/log/mysql/mysql-bin.log

$ sudo systemctl restart mysql

$ mysql -u root -p

mysql> CHANGE MASTER TO MASTER_HOST='192.168.56.30', MASTER_USER='myslave', MASTER_PASSWORD='mysecret', MASTER_LOG_FILE='mysql-bin.000002', MASTER_LOG_POS=613;

mysql> START SLAVE;

# 其他

# MySQL 表加锁
mysql> FLUSH TABLES WITH READ LOCK;
# MySQL 表解锁
mysql> UNLOCK TABLES;
# 导出数据库的方法
$ mysqldump -u root -p --all-databases --master-data > dbdump.sql
# 发送文件给目标主机
$ scp dbdump.sql 192.168.56.31:/tmp
# 导入数据文件
$ mysql -u root -p < /tmp/dbdump.sql
# 虚拟机中解决从服务器与主服务器有相同 MySQL 序号
$ sudo rm /var/lib/mysql/auto.cnf

具体表现为 START SLAVE; 时出现 master and slave have equal MySQL server UUIDs 错误，原因是直接复制虚拟机导致的。
