    # 在 Ubuntu 安装 LNMP 1.4 使用 Laravel 5.4

    日期：2018-02-26

    # 配置环境

    进入目录 ~/lnmp1.4-full/src

    解压已经安装的 PHP 版本 tar -xjf php-7.1.7.tar.bz2

    进入拓展目录 ~/lnmp1.4-full/src/php-7.1.7/ext/fileinfo

    在当前目录依次执行

    /usr/local/php/bin/phpize

    ./configure --with-php-config=/usr/local/php/bin/php-config

    make && make install

    进入目录 /usr/local/php/lib/php/extensions

    进入类似目录 no-debug-non-zts-20160303

    检查有无文件 fileinfo.so

    进入文件 /usr/local/php/etc/php.ini

    查找 extension_dir 再下面一行添加上 extension = "fileinfo.so"

    查找 disable_function 删除 proc_open , proc_get_status

    退出并保存配置文件

    进入文件 /usr/local/nginx/conf/fastcgi.conf

    使用 # 注释此行 fastcgi_param PHP_ADMIN_VALUE "open_basedir=$document_root/:/tmp/:/proc/";

    退出并保存配置文件

    进入虚拟主机配置文件 /usr/local/nginx/conf/vhost/<域名>.conf

    注意网站根目录应为 root /home/wwwroot/<域名>/public;

    注释创建虚拟机时选的 other URL 重写配置，并写上 Laravel 官方提供的 URL 重写规则

    #include other.conf;
location / {
    try_files $uri $uri/ /index.php?$query_string;
}

    退出并保存配置文件

    重启 LNMP lnmp restart

    # 获取文件

    进入目录 /home/wwwroot

    执行命令 git clone git@github.com:<用户名>/<仓库名>.git

    复制文件到网站目录

    cp -rf <仓库名>/* <域名>/

    进入网站目录 cd <域名>

    赋予 Laravel 项目的 storage 目录权限 chown -R www:www storage/

    安装依赖 执行 composer install
