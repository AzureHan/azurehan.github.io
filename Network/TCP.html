# TCP 协议

# 1. TCP 连接

# 1.1 建立连接

双方三次握手建立连接 :

  1. 客户端发送数据包 [ SYN = 1 , seq = x ] 并进入 SYN-SENT 状态

  2. LISTEN 状态的服务器接收到数据包 , 返回数据包 [ SYN = 1 , ACK = 1 , seq = y , ack = x ] 并进入 SYN-REVD 状态

  3. 客户端接收到确认数据包 , 发送数据包 [ ACK = 1 , seq = x + 1 , ack = y + 1 ] 并进入 ESTABLISHED 状态 , 服务器接收到这个数据包也进入 ESTABLISHED 状态 .

seq = sequence number ( 序列号 )

ack = acknowledgement ( 确认号 )

ESTABLISHED = 既定 ( 是指已经确定 )

简单来说 : 客户端请求服务器建立连接、服务器同意请求并要求客户端确认、客户端确认进行连接。

# 1.2 传输数据

# 1.2.1 滑动窗口以及拥塞控制

TCP协议作为一个可靠的面向流的传输协议，其可靠性和流量控制由滑动窗口协议保证，而拥塞控制则由控制窗口结合一系列的控制算法实现。

# 1.3 断开连接

双方四次挥手断开连接 :

  1. 请求方发送数据包 [ FIN = 1 , seq = u ] 进入 FIN-WAIT-1 状态

  2. 被请求方接收到数据包并返回确认数据包 [ ACK = 1 , seq = v , ack = u + 1 ] ，请求方进入 FIN-WAIT-2 状态

  3. 被请求方发送数据包 [ FIN = 1 , ACK = 1 , seq = w , ack = u + 1 ] 并且最终中断连接

  4. 请求方接收到数据包并返回确认数据包 [ ACK = 1 , seq = u + 1 , ack = w + 1 ] 并且进入 TIME-WAIT 状态、等待 2MSAL 时间后最终中断连接

简单来说 : 请求方 ( 不需要发送数据 ) 请求断开连接、被请求方同意断开请求、然后被请求方也表示 ( 不需要发送数据 ) 最终断开连接、请求方再请求试探被请求方、没有回应也就确认最终断开。
